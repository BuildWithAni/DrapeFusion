## ✨ Features

<div align="center">
  
  | Feature | Description |
  |:--------|:------------|
  | 🎯 **AI Virtual Try-On** | Photorealistic garment-to-model transfer using **IDM-VTON (SDXL)** |
  | 🆓 **Free Tier** | Powered by **Hugging Face ZeroGPU** — no API key needed to start |
  | ☁️ **Paid Fallback** | Automatic upgrade to **Replicate API** for production reliability |

## 🚀 Quick Start (No API Keys Needed)

The app works **fully functional** with mocked data — no API keys, no database, no setup required.

```bash
# 1. Navigate to the Next.js app
cd drapefusion

# 2. Install dependencies
npm install

# 3. Start the dev server
npm run dev
```

> **Open [http://localhost:3000](http://localhost:3000)** — sign up and start generating in seconds.

## 🧠 How It Works

<details>
<summary><strong>🔹 The VTON Pipeline (3 Engines)</strong></summary>
<br/>

DrapeFusion uses a **smart fallback architecture** with three VTON engines:

```
User uploads garment + model photo
            │
            ▼
┌─────────────────────────────┐
│  Engine 1: Hugging Face     │ ◄── FREE, no API key
│  yisol/IDM-VTON (ZeroGPU)   │
└──────────┬──────────────────┘
           │ (quota exceeded / timeout)
           ▼
┌─────────────────────────────┐
│  Engine 2: Replicate API    │ ◄── PAID, production grade
│  cuuupid/idm-vton           │
└──────────┬──────────────────┘
           │ (insufficient credits)
           ▼
┌─────────────────────────────┐
│  Engine 3: Local Gradio     │ ◄── SELF-HOSTED, your GPU
│  local-vton/app.py          │
└──────────┬──────────────────┘
           │ (all servers fail)
           ▼
┌─────────────────────────────┐
│  Canvas Fallback (Browser)  │ ◄── ALWAYS WORKS
│  Client-side compositing    │
└─────────────────────────────┘
```

</details>

<details>
<summary><strong>🔹 Local VTON Pipeline (Full Detail)</strong></summary>
<br/>

When running locally, here's what happens under the hood:

1. **🧍 Pose Estimation** — OpenPose detects body keypoints
2. **👕 Human Parsing** — SCHP/AugmentCE2P segments clothing regions
3. **🧬 DensePose** — Detectron2 maps body parts with UV coordinates
4. **🎭 Auto-Masking** — Generates a precise binary mask of the garment area
5. **✂️ Auto-Crop** — Crops/resizes model to 3:4 aspect ratio
6. **🎨 IP-Adapter Conditioning** — Encodes the garment image into the diffusion model
7. **🖌️ SDXL Inpainting** — 30 denoising steps at guidance scale 2.0
8. **🖼️ Output** — 768×1024 photorealistic try-on image

</details>

## 🛠️ Full Setup

### Prerequisites

- **Node.js** 20+
- **Python** 3.10+ (for local VTON)
- **Git**

### Environment Variables

Create `drapefusion/.env.local`:

```env
# ─── AI Generation ───────────────────────────
REPLICATE_API_TOKEN=your_replicate_token
HF_TOKEN=hf_your_huggingface_token

# ─── Image Hosting ───────────────────────────
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

# ─── Local VTON ──────────────────────────────
USE_LOCAL_VTON=false       # Set true to use your local GPU
```

### Running the Local VTON Server

```bash
# 1. Install Python dependencies
cd local-vton
pip install -r requirements.txt

# 2. Download missing model weights (~1.3 GB)
python download_models.py

# Or download the full model repo (~7 GB)
python ../download_model.py

# 3. Install Detectron2 + DensePose
#    Follow: https://github.com/facebookresearch/detectron2
#    Follow: https://github.com/facebookresearch/densepose

# 4. Start the Gradio server
python app.py
# → Opens at http://127.0.0.1:7860

# 5. Set USE_LOCAL_VTON=true in .env.local
# 6. Restart your Next.js dev server
```

### Production Build

```bash
cd drapefusion
npm run build
npm start
```
